# Hydra Terra — WebGPU fluid & erosion sandbox

A 3D "powder toy" style sandbox: procedurally generated island terrain, real-time
shallow-water fluid simulation, and hydraulic + thermal erosion — all running on
the GPU via WebGPU. Click anywhere to pour water and watch it carve channels,
carry sediment, and build deltas.

Note: This project was completed as a capability test of Claude Fable 5.

## Run

WebGPU needs a secure context, so serve the folder over localhost:

```sh
cd fluid-powder-toy
python3 -m http.server 8000
```

Then open <http://localhost:8000> in a WebGPU-capable browser
(Chrome/Edge 113+, Safari 18+, Firefox 141+).

## Controls

| Input          | Action                          |
| -------------- | ------------------------------- |
| Left-drag      | Pour water at the cursor        |
| Right-drag     | Orbit the camera                |
| Mouse wheel    | Zoom                            |
| `R` / button   | Generate a new terrain          |
| `C` / button   | Drain all water                 |
| Sliders        | Flow rate and brush radius      |

## How it works

Everything lives in two WGSL modules inside `main.js`; the simulation never
reads back to the CPU.

- **Terrain** — ridged fbm with domain warp and a radial island falloff,
  generated by a compute pass into a 256×256 heightfield.
- **Water** — the virtual-pipes shallow-water model (Mei et al. 2007):
  per-cell outflow flux in 4 directions, depth integration, and a velocity
  field, run 3 substeps per frame.
- **Erosion** — sediment capacity from slope × speed × depth; terrain
  dissolves into suspended sediment when under capacity and deposits when
  over, with a relief clamp against the 4-neighbour mean to keep the
  feedback loop from checkerboarding. Sediment is advected
  semi-Lagrangially with the flow; a thermal-erosion pass relaxes slopes
  past the talus angle.
- **Picking** — a single-thread compute raymarch writes the cursor hit
  point to a small GPU buffer that the pour pass and the cursor-ring shader
  consume directly (zero CPU readback).
- **Rendering** — heightfield meshes generated from `vertex_index`
  (sky / terrain / water passes, 4× MSAA): soft-shadow heightfield
  raymarch, procedural sky with sun and fbm clouds, depth-absorbing fresnel
  water with sediment tinting and flow foam, erosion scars and silt
  deposits tinted from the terrain-vs-original delta, ACES tonemapping.

## Files

- `index.html` — UI shell (panel, sliders, hints, FPS counter)
